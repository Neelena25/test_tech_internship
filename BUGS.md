# BUGS.md

## 1. Сохранить объявление

### BUG-01 — Отрицательные поля принимаются сервисом

##### Описание:
Сервис принимает отрицательное значение полей с целочисленным типом и возвращает HTTP 200

##### Шаги:

###### 1. Отправить POST baseUrl/api/1/item с отрицательным значением поля sellerID
###### 2. Отправить POST baseUrl/api/1/item с отрицательным значением поля price
###### 3. Отправить POST baseUrl/api/1/item с отрицательным значением поля likes
###### 4. Отправить POST baseUrl/api/1/item с отрицательным значением поля viewCount
###### 5. Отправить POST baseUrl/api/1/item с отрицательным значением поля contacts

##### Фактический результат:
- HTTP 200
- Ответ:
{
    "status": "Сохранили объявление - {уникальный идентификатор}"
}

##### Ожидаемый результат:
- HTTP 400
- Сообщение о некорректном значении

## 2. Получение всех объявлении по sellerID (GET baseUrl/api/1/{id}/item)

### BUG-02 — Отрицательный sellerID принимается сервисом

##### Описание:
Сервис принимает отрицательное значение sellerID и возвращает HTTP 200 с пустым массивом, вместо ошибки валидации.

##### Шаги воспроизведения:
###### 1. Выполнить GET /api/1/sellerID/item с sellerID : -488283

##### Фактический результат:
- HTTP 200
- Ответ: []

##### Ожидаемый результат:
- HTTP 400
- Сообщение о некорректном значении sellerID
